MAF advanced preferences

The default settings in effect when you install MAF for the first time should be enough to allow correct loading and saving of MAF and MHTML files. To customize advanced aspects of page loading and archiving, the extension's preferences dialog can be accessed from the Tools » Mozilla Archive Format » Preferences menu item.

The following screenshots show the preferences dialog of a previous version of MAF on Linux.

Temporary folder

Location of the temporary directory used by MAF to extract archive files. Currently defaults to the maf/maftemp subdirectory of the user profile's directory.

Clear Temporary folder when browser exits

This option is enabled by default. If disabled, the contents of the temporary directory used by MAF to extract archive files is preserved after the browser exits, and must be cleaned up manually.

Open Archive Mode

There are three open modes available. These configure how MAF behaves once an archive is opened.

  1. Show open dialog box - Displays a dialog of all the open archives in a tree form, you select which files you would like displayed in tabs.
  2. Open all HTML in new tabs (default) - Opens all resources in new tabs. If your archive is big, this could cause opening the archive to take a long time.
  3. Do nothing - Allows you to open as many archives as you want and browse to display any of the saved resources after complete.
Save Extended Meta-Data

With this preference enabled, additional page information such as history, text zoom and scroll position is saved for each page. There is currently no preference to restore this saved information as yet.

Use alternative save component

With this preference enabled, a different component is used by MAF to serialize HTML pages to disk. This component saves images linked in CSS pages but it currently does NOT work on framed pages.

Alert on single archive complete

With this preference enabled, an alert box is displayed each time a single page is saved in an archive. This is not applicable if multiple tabs are being saved in an archive at once.

Rewrite Archive Document URLs

With this preference enabled, the DOM tree of expanded archive HTML files would be processed as they finish loading in tabs. The processing would iterate over all the links in the page and replace absolute links to resources in the page with local resources in the archive (if possible). This would allow users to browse linked pages in an archive seamlessly.

Enable MAF Protocol
Enable document.write override

These preferences control features that are not available in the upcoming official release of MAF for Firefox 3. For reference, you can see the old documentation still accessible at the Internet Archive.
