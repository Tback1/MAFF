What's the difference?

The version on the Firefox Add-ons site is updated less frequently. Therefore, such updates have been tested more extensively.

If you download Mozilla Archive Format from this site, your installation will be automatically updated to the latest version as soon as it is available.

Before release, the latest version is tested on Firefox 3 under Windows XP, but is expected to work on all the platforms supported by Firefox 3.

What's new in the latest version?

MAF has the following user-visible improvements over MAF :

  • A new dialog for conversion of previously saved pages is now available. See the related documentation for details.
  • New preference to use the page title instead of the default suggested file name in the "Save As" dialog. Enabling this preference provides functionality comparable to the File Title and Title Save extensions.
  • The order of the tabs is now preserved when opening and saving multi-page MAFF archives.
  • Now the character set specified in MAFF archives is always enforced, and a different character set cannot be selected.
  • In the multiple tab selection dialog, occasionally it was not possible to change the current selection after some time passed.
  • Improvements to the integrated Save Complete extension:
    • Improved performance when saving pages containing many images or other content.
    • Improved reliability in handling references to images or other content.
    • Errors that occur while downloading the main file are now reported correctly.
    • Pages that are contained in an archive are now saved correctly.
    • When the save operation fails, an empty file isn't created anymore in place of the target file.

MAF has the following known issues:

  • In order to open MHTML files when the IE Tab extension is installed, the filter /^file:\/\/\/.*\.(mht|mhtml)$/ must be disabled from the IE Tab options.
  • The Download Sort extension has a known incompatibility with MAF. The conflict cannot be resolved without proper modifications to both extensions, and Download Sort is not actively maintained at this time.
  • When saving a page in an archive fails, retrying the download from the Downloads window does not achieve the expected result.
  • Opening very long documents contained inside MHTML files may cause the browser to stop responding temporarily.
  • With versions of Firefox prior to 3.5.3, web archive files must have a lowercase file name extension in order to be opened.
